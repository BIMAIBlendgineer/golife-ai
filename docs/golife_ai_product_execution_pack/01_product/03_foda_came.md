# FODA y CAME

## FODA

### Fortalezas
- Seis dominios funcionales.
- LifeGraph común.
- Home Today como interfaz simple.
- IA coordinadora.
- Local-first.
- Backend operacional con PostgreSQL.
- Admin Next.js.
- Clean-room strategy.
- OpenRouter preparado.
- Métrica norte: misiones útiles completadas por usuario activo.

### Oportunidades
- Crear categoría "Life OS con IA".
- Unir productividad, dinero, comida, hábitos y armario.
- Captar usuarios cansados de múltiples apps.
- Usar diario para personalización profunda.
- Crear reportes semanales de valor real.
- Medir coste y utilidad con admin.

### Debilidades
- Scope amplio.
- UX puede volverse compleja.
- Dominios aún necesitan profundidad.
- Parser multi-evento pendiente.
- Misiones deben accionar entidades reales.
- Safety emocional pendiente.
- Privacidad por evento/cifrado pendientes.
- Reorganización de repo necesaria.

### Amenazas
- Riesgo legal por licencias si se copia código.
- Competidores pueden copiar features aisladas.
- Apps verticales tienen más profundidad.
- Recomendaciones malas erosionan confianza.
- Coste IA puede subir.
- Diario/reflexión puede confundirse con terapia.

## CAME

### Corregir
- Reorganizar repo.
- Completar dominios por fases.
- Crear parser multi-evento.
- Hacer misiones accionables.
- Implementar privacidad por evento.
- Crear safety emocional.
- Simplificar UX alrededor de Today/Capture/Board/Review.

### Afrontar
- Mantener clean-room.
- No copiar GPL.
- Proteger diario localmente.
- Explicar límites de IA.
- Controlar coste IA por endpoint y usuario.
- Usar feature flags.

### Mantener
- LifeGraph.
- 3 misiones.
- Risk engine.
- Feedback learning.
- Admin operacional.
- PostgreSQL.
- CI.
- Explicación con evidencia.

### Explotar
- Diario como memoria humana.
- Recipe Rescue como ahorro inmediato.
- Kanban como control visual.
- QuickCal como realismo semanal.
- ClosetLess como anti-consumo.
- Weekly Review como retención.
