# PDR — Reorganización del monorepo

## Objetivo
Implementar Reorganización del monorepo como una etapa verificable del producto GoLife AI.

## Problema
El usuario necesita que esta capacidad reduzca fricción, mejore claridad y alimente LifeGraph.

## Requisitos funcionales
- apps/mobile_flutter
- apps/admin_next
- services/ai_gateway
- services/web_backend
- docs
- packages/contracts
- README raíz
- CI actualizado
- docs/archive
- REPO_REORGANIZATION_MAP

## Requisitos no funcionales
- Privacidad explícita.
- Tests.
- Telemetría cuando aplique.
- Fallback.
- Documentación.
- No introducir payload sensible innecesario.

## Criterios de aceptación
- Flujo funcional de extremo a extremo.
- Tests verdes.
- UI usable.
- Contratos documentados.
- No rompe CI.
