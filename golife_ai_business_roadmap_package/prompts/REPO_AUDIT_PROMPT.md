# Prompt — Repository Audit

Analyze the six reference repositories in `references/`.

For each repo, produce:

- name;
- URL;
- branch;
- stack;
- app architecture;
- data models;
- screens;
- reusable concepts;
- legal license;
- copy/adapt/inspiration recommendation;
- risks.

Then create:

- `REPO_AUDIT.md`;
- `LICENSE_MATRIX.md`;
- `DEPENDENCY_MATRIX.md`;
- `FEATURE_EXTRACTION_MATRIX.md`.

Do not write application code.
