# 06 — CAME

## Corregir debilidades

### Debilidad: producto demasiado amplio

Acción:

- lanzar solo 4 dominios en MVP: tareas, hábitos, dinero manual, despensa manual;
- armario entra en fase 2;
- no incluir banco ni OCR avanzado al inicio.

### Debilidad: recomendaciones superficiales

Acción:

- usar reglas deterministas + IA;
- exigir explicación con evidencia;
- medir si el usuario aceptó o rechazó la recomendación.

### Debilidad: coste LLM

Acción:

- cachear recomendaciones;
- usar modelos baratos para clasificación;
- usar modelos mejores solo para síntesis semanal;
- permitir provider swap.

### Debilidad: dependencia de datos manuales

Acción:

- captura rápida por texto/voz;
- plantillas;
- importación simple CSV;
- onboarding mínimo.

## Afrontar amenazas

### Amenaza: copia por grandes plataformas

Acción:

- crear marca fuerte;
- centrarse en LifeGraph y explicación;
- construir comunidad early adopter;
- documentar un método propio.

### Amenaza: privacidad

Acción:

- local-first cuando sea posible;
- cifrado;
- exportación/borrado de datos;
- privacy dashboard.

### Amenaza: licencias open-source

Acción:

- `LICENSE_MATRIX.md` obligatorio;
- separar inspiración de implementación;
- clean-room si se busca producto cerrado;
- usar MIT/Apache/BSD si se reutiliza código directo.

## Mantener fortalezas

### Fortaleza: uso diario

Acción:

- push matinal con 3 misiones;
- check-in nocturno;
- feedback de utilidad.

### Fortaleza: diferenciación cruzada

Acción:

- cada recomendación debe conectar al menos dos dominios cuando sea posible.

Ejemplo:

> “No compres comida hoy: tienes 3 alimentos próximos a vencer y tu presupuesto semanal está al 82%.”

## Explotar oportunidades

### Oportunidad: ahorro + anti-desperdicio

Acción:

- crear campaña: “7 días tirando menos comida”.
- crear informe semanal de dinero ahorrado estimado.

### Oportunidad: familia

Acción:

- lanzar plan Family después de validar MVP individual.
