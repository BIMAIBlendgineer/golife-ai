# 22 — Operating Model

## Roles mínimos

- Product owner.
- Flutter developer.
- Backend/AI developer.
- UX designer.
- QA.
- Legal/privacy reviewer.

## Cadencia

- weekly build;
- daily issue triage;
- demo cada viernes;
- beta feedback cada semana.

## Repos recomendados

```text
golife/
  app/
  ai-gateway/
  docs/
  schemas/
  infra/
```

## Entornos

- local;
- staging;
- beta;
- production.

## Definition of Done

Cada feature debe tener:

- test;
- schema;
- UX copy;
- fallback sin IA;
- privacy review;
- analytics event;
- error state;
- documentation.
