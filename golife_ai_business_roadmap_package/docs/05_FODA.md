# 05 — FODA / SWOT

## Fortalezas

1. Idea clara: una IA que coordina vida diaria, no una app aislada.
2. Uso frecuente: tareas, comida, dinero y hábitos ocurren todos los días.
3. Diferenciación: cruza dominios que normalmente están separados.
4. MVP posible: se puede empezar con captura rápida + misiones.
5. Escalable: nuevos dominios pueden entrar como módulos.
6. Narrativa comercial potente: “menos caos, menos desperdicio, menos gasto”.

## Oportunidades

1. Saturación de apps aisladas: el usuario busca simplificación.
2. Crecimiento de IA como interfaz natural.
3. Interés por ahorro y anti-consumo.
4. Mayor sensibilidad sobre desperdicio alimentario.
5. Posible integración futura con calendario, banco, supermercados y wearables.
6. Posible expansión familiar.

## Debilidades

1. Producto demasiado amplio si no se controla.
2. Riesgo de recomendaciones superficiales.
3. Coste LLM si se abusa de IA.
4. Dependencia de datos introducidos por el usuario.
5. Complejidad UX: seis dominios pueden confundir.
6. Riesgo legal si parece asesor financiero, médico o psicológico.

## Amenazas

1. Grandes plataformas pueden copiar funciones.
2. Usuarios pueden desconfiar de dar datos personales.
3. App stores pueden exigir políticas de privacidad estrictas.
4. Costes de proveedor IA pueden variar.
5. Licencias open-source incompatibles si se copia código sin auditoría.
6. Recomendaciones erróneas pueden generar pérdida de confianza.

## Conclusión FODA

GoLife AI tiene potencial alto si se mantiene como **coordinador diario** y no como “mega app desordenada”.

La función inicial debe ser:

> Misiones diarias generadas desde tareas, hábitos, dinero y despensa.
